# Test_cases


## Тест-кейс №1: Успешная регистрация нового пользователя
​​
Предусловие: Пользователя с таким email нет в базе данных.
​

Шаги:


​1. Отправить POST запрос на /api/register.

​
2. Передать в теле запроса уникальные данные (новый email, password, username).
​

### Ожидаемый результат: Статус 201 Created. Пользователь успешно создан в базе данных.
​

## Тест-кейс №2: Регистрация с уже существующим email
Предусловие: Пользователь с данным email уже есть в базе данных.
​

Шаги:
​

1. Отправить POST запрос на /api/register.


2. Использовать email пользователя, который уже существует.
​
### Ожидаемый результат: Статус 409 Conflict (или 400 Bad Request, зависит от логики бэкенда). В ответе сообщение: "message": "User already exists".
​

## Тест-кейс №3: Невалидный формат email
​Предусловие: Нет.
​

Шаги:

​
1. Отправить POST запрос на /api/register.
​

2. В поле email ввести значение без символа @ или доменной зоны (например, testmail.com).

### ​Ожидаемый результат: Статус 400 Bad Request. Система возвращает ошибку валидации поля email.
